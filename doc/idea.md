# 전지적 지구 시점 (Omniscient Earth View) — 서비스 기획 리포트

## 1. 서비스 개요
전지적 지구 시점은 전 세계에서 발생하는 각종 사건·사고·이슈(테러, 재난, 시위, 경제/산업 이벤트, 엔터테인먼트 일정 등)를 “지구본 기반 인터랙티브 지도” 위에서 실시간에 가깝게 탐색할 수 있게 하는 글로벌 이슈 관측 플랫폼이다.

사용자는 텍스트 중심의 뉴스 소비를 넘어, 사건이 발생한 지리적 맥락(대륙/국가/도시/인근 지형, 주변 국가와의 거리, 인구 밀도 등)을 즉시 시각적으로 이해하며, 타임라인으로 확산·연결 관계를 따라가듯 탐색한다.

## 2. 문제 정의 및 접근
### 2.1 문제 정의
- 뉴스는 대부분 텍스트/이미지 중심으로 제공되어, “위치에 대한 체감”이 부족하다.
- 특정 사건(예: LA 대형 화재, 중동 분쟁, 해상 사고 등)의 규모가 “어디에서 얼마나 큰지”가 직관적으로 전달되지 않아, 이해/공감/기억이 약해진다.
- 과거 사건을 구조적으로 탐색하기 어렵다(연도·지역·유형별 비교, 군집/패턴 관찰의 어려움).

### 2.2 해결 접근(핵심 관점)
- “지구본(Globe)”을 기본 인터페이스로 삼아, 모든 이슈를 좌표와 시간축으로 정렬한다.
- AI가 뉴스 헤드라인을 수집·분류하고, 위치 후보 좌표를 추정해 사건을 지도에 올린다.
- 사용자는 카테고리/시간/지역 필터를 통해 지구 단위로 탐색하고, 클릭 시 카메라가 해당 지역으로 회전·줌인하여 맥락 중심의 이해를 제공한다.

## 3. 핵심 사용자 경험(UX) 컨셉
### 3.1 기본 화면
- 중앙: 3D 지구본(또는 2.5D 지구 투영) + 사건 마커/클러스터 + 시간 슬라이더
- 좌측/하단 패널: 카테고리별 헤드라인 피드(실시간/일간)
- 우측 패널: 선택된 사건의 요약 카드(핵심 문장, 출처, 관련 기사 묶음, 타임라인, 신뢰도/근거)

### 3.2 상호작용(클릭/탐색)
- 헤드라인 클릭 → 지구본이 해당 좌표로 회전 → 단계적 줌인(국가→도시→현장 인근) → 사건 시각화 오버레이 표시
- 마커 클릭 → 사건 카드 노출 + 관련 기사/영상/공식 발표 링크 + 유사 사건 탐색(같은 유형/지역/기간)

### 3.3 “지구 시점” 연출 요소
- 거리/스케일 체감: 반경 오버레이(예: 화재 영향권, 지진 진원과 파급, 비행/이동 경로)
- 군집/확산 시각화: 기간별 발생 밀집도 히트맵, 라인(경로), 파동(영향)
- 타임라인 재생: 특정 기간을 재생하면 사건이 시간순으로 지구본 위에 “등장”한다.

## 4. 핵심 기능 정의
## 4.1 실시간 글로벌 이슈 지도
- 전 세계 사건·사고·이슈를 지도(지구본) 상에서 마커로 표시
- 스케일에 따라 클러스터링(도시 단위 확대 시 개별 사건 분리)
- 기본 탐색 범위: 최근 7일(디폴트)

## 4.2 AI 기반 뉴스 수집/분류/좌표화
- 뉴스/공식 발표/기관 공지 등 다양한 소스에서 헤드라인 수집
- 언어 감지 및 다국어 처리(원문+자동 번역 요약 제공 가능)
- 분류 체계(예시):
  - 재난(화재/홍수/태풍/지진/산사태)
  - 사건사고(교통/항공/해상/폭발)
  - 분쟁/테러/치안
  - 정치/외교
  - 산업/기술(제품 발표회, 기업 이벤트)
  - 보건(감염병, 공중보건 경보)
  - 문화/엔터테인먼트(콘서트/투어/시상식)
- 기사에서 장소를 추출하고 좌표 후보를 생성(지오코딩/가제터 기반)

## 4.3 카테고리 피드 + 지구본 연동
- 화면 한 영역에 카테고리별 헤드라인 리스트 제공
- 리스트 클릭 시 지구본 카메라 이동(회전→줌)
- 선택 사건 기준으로 주변 관련 사건/연관 키워드도 자동 제안

## 4.4 사건 상세(컨텍스트 카드)
- 핵심 요약(1~3문장), 발생 시각/지역, 출처(링크)
- 관련 기사 묶음(동일 사건으로 추정되는 여러 매체 기사)
- 위치 컨텍스트: 행정구역, 주변 주요 도시, 거리 스케일(예: “도심에서 약 N km”)
- 타임라인: 최초 보도→추가 보도→종결/후속 조치

## 4.5 과거 데이터 아카이브(리플레이)
- 예: “지난 100년 테러 사건”을 조건으로 검색/리스트업
- 지구본에 시간순으로 표시하고, 필터(연도/대륙/국가/유형/피해 규모 등) 제공
- 기간을 재생하면 사건이 시간축으로 재현(히스토리 모드)

## 4.6 대중문화/이벤트 타임라인(엔터테인먼트 모드)
- 아티스트/팀 단위로 “투어/콘서트/공연/행사”를 좌표+시간으로 시각화
- 예:
  - 마이클 잭슨: 연도별 투어 도시가 지구본에 점으로 찍히고, 일정 흐름이 라인으로 연결
  - BTS: 월드투어 일정이 도시별로 표시되고, “다음 달 어디에서 한다”는 기사/공지가 발생하면 미래 이벤트로 표시
- 글로벌 유저가 동일한 콘텐츠를 지구본 위에서 탐색하며 공유 가능

## 5. 데이터 파이프라인(수집→정제→표현)
### 5.1 수집(Ingestion)
- 소스 후보:
  - RSS/공식 API 제공 언론
  - 기관 발표(재난/기상/보건/항공/해양 관련 기관)
  - 기업 이벤트(공식 뉴스룸, 일정 공지)
- 수집 주기:
  - 헤드라인: 수분~수십분 단위
  - 정제/그룹핑: 배치(예: 10~30분) + 이벤트 발생 시 온디맨드

### 5.2 정제(Normalization)
- 중복 제거: 동일 기사/재배포 기사 정규화
- 사건 단위 그룹핑: 여러 기사 → 하나의 “이벤트 엔티티”로 묶기
- 메타데이터 표준화:
  - 시간(현지/UTC), 국가/도시/행정구역
  - 카테고리, 키워드, 출처 신뢰 메타(출처 유형 등)

### 5.3 위치 추정(Geocoding)
- 기사 텍스트에서 지명/시설명/좌표 표현 추출(NER)
- 지오코더로 좌표 후보 생성:
  - 도시/행정구역 중심 좌표
  - 특정 시설(공항/경기장/컨벤션센터) 좌표
- 후보 좌표 스코어링:
  - 텍스트 근거(지명 빈도, 문장 내 역할)
  - 사건 유형과 장소 적합도(예: “발표회”는 컨퍼런스/본사/행사장)
  - 과거 동일 키워드의 위치 히스토리

### 5.4 요약/분류(LLM/ML)
- 헤드라인/본문 요약(짧은 요약 + 핵심 키워드)
- 다국어 매핑(원문 언어 보존 + 사용 언어로 요약 제공)
- 사건 타입 분류 및 태깅(멀티라벨)

### 5.5 서빙(Serving)
- 사건 피드 API: 시간순/카테고리/지역 바운딩 박스 기반
- 지도 타일/3D 씬 최적화:
  - 줌 레벨별 클러스터링 결과 제공
  - 최근 7일 데이터는 캐시/프리컴퓨트

## 6. 지도 시각화 설계(표현 방식)
- 점(Points): 단일 사건 마커, 색상=카테고리, 크기=영향/언급량(옵션)
- 클러스터(Clusters): 축소 시 지역별 밀집도 표시
- 히트맵(Heatmap): 기간 내 다발 지역 강조
- 라인/경로(Lines): 투어/행사 이동 경로, 확산 경로(옵션)
- 반경/영역(Polygons/Rings): 영향권, 대피 구역, 기상 범위(데이터 가능 시)

## 7. 대표 사용자 시나리오(예시)
### 7.1 LA 대형 화재 이해 흐름
1) 사용자가 “재난 > 화재” 카테고리를 선택한다.  
2) 피드에서 “Los Angeles wildfire expands …” 헤드라인을 클릭한다.  
3) 지구본이 북미 서부로 회전하고, 캘리포니아 남부로 단계적으로 줌인한다.  
4) 지도 위에 화재 발생 추정 지점이 표시되고, 영향 반경/연기 확산(가능 데이터 기반)이 오버레이된다.  
5) 사건 카드에서 요약과 함께 관련 기사 묶음을 확인하고, “지난 5년간 동일 지역 대형 화재”로 히스토리 탐색을 이어간다.

## 8. 탐색/필터링 요구사항
- 기본 필터:
  - 기간(기본 7일, 24시간/30일/커스텀)
  - 카테고리/태그
  - 지역(대륙/국가/도시, 지도 드래그로 영역 지정)
- 고급 필터(확장):
  - 출처 유형(공식/언론/기타)
  - 언급량/확산 속도
  - 이벤트 상태(진행/종결/후속)

## 9. 플랫폼 구성 요소(제품 관점)
- 지구본 뷰어(3D)
- 사건/이슈 피드(카테고리별)
- 사건 상세 페이지(요약/출처/타임라인/관련 기사)
- 아카이브(과거 사건 검색 + 재생)
- 엔터테인먼트 모드(아티스트/투어/이벤트 타임라인)
- 공유 기능(링크/임베드/스크린샷 기반 공유)

## 10. 데이터/저작권/출처 표기 원칙(설계 항목)
- 기사 원문은 출처 링크 중심으로 연결하고, 플랫폼 내에서는 요약/메타데이터 중심으로 제공
- 모든 사건 카드에 출처(매체/기관/발표 페이지) 명시
- 임베드/미리보기는 소스 정책(robots, API TOS)에 따라 제한 가능하도록 설계

## 11. MVP 범위 제안(초기 구현 단위)
- 최근 7일 데이터 중심
- 3~5개 카테고리(재난/사건사고/분쟁/산업이벤트/문화이벤트)
- 뉴스 수집(제한된 소스) + 기본 요약 + 기본 지오코딩(도시 단위)
- 지구본 클릭/피드 클릭 연동 + 사건 상세 카드 + 관련 기사 링크
- 간단한 아카이브(최근 1~3년 일부 데이터부터)

## 12. 확장 로드맵 아이템(추가 기능 후보)
- 고정밀 위치(시설/교차로 수준) + 근거 하이라이트(“좌표 추정 근거 문장” 표시)
- 사건 간 연관 그래프(키워드/지역/시간 기반 연결)
- 알림(사용자 관심 지역/키워드 기반)
- 커뮤니티 레이어(사용자 제보/정정/태그 보강, 검증 플로우 포함)
- 공개 API/임베드 위젯(외부 사이트에 지구본 사건 뷰 삽입)

## 13. 용어 정의(간단)
- 이벤트 엔티티(Event): 여러 기사를 하나로 묶은 “사건 단위” 데이터 모델
- 좌표 후보(Candidate Coordinates): 기사에서 추정된 1개 이상 위치 후보와 스코어
- 리플레이(Replay): 특정 기간 사건을 시간축으로 재생하는 탐색 모드
